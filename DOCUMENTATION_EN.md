# DiploChain V2 — Full Project Documentation

## 1. Project Overview
DiploChain is a decentralized platform for issuing, verifying, and revoking academic diplomas using **Hyperledger Fabric**. It ensures the immutability of academic records while providing a real-time audit dashboard for administrators.

**Key Features:**
- **Immutability**: Every diploma is hashed and stored on a private blockchain ledger.
- **Verification**: Third parties can instantly verify a diploma's authenticity via hash checks.
- **Revocation**: Institutions can revoke diplomas in case of fraud or error.
- **Real-time Audit**: A dynamic dashboard monitoring network health and transaction flow.

```text
| (Orderer, Peer, CA, CouchDB)|

```

### Components Interaction
1.  **Frontend**: React client polls the API server for stats and triggers transactions.
2.  **API Server**: Central bridge. Handles DB queries for history and executes Fabric CLI commands inside containers for blockchain operations.
3.  **Fabric Network**: Executes GOLANG chaincode and maintains the world state in CouchDB.
4.  **PostgreSQL**: Stores metadata, audit logs, and user sessions for fast retrieval.

## 3. Installation Guide

### Prerequisites
- Docker & Docker Compose
- Node.js (v18+)
- Golang (v1.21+)
- WSL2 (if on Windows)

### Setup Steps
1.  **Clone the repository**:
    ```bash
    git clone <repository_url>
    cd diplochain
    ```
2.  **Start the Infrastructure**:
    ```bash
    ./run_diplochain.sh
    ```
3.  **Initialize Backend**:
    ```bash
    cd fabric-api-server
    npm install
    npm start
    ```
4.  **Initialize Frontend**:
    ```bash
    cd audit-dashboard
    npm install
    npm run dev
    ```

## 4. Environment Variables
Project-wide `.env` (in `fabric-network/.env`):
- `COUCHDB_PASSWORD`: Password for CouchDB instances.
- `FABRIC_VERSION`: e.g., `2.5`.
- `CHAINCODE_VERSION`: Current stable version (`1.3`).
- `ORDERER_ADDRESS`: GRPC address for the orderer.

  - Body: `{ "fn": "RegisterDiploma", "args": ["D-123", "Hash..."] }`

## 6. Blockchain Logic

### Chaincode Functions (GOLANG)
- **RegisterDiploma**: Serializes diploma metadata and stores it with `PutState`.
- **VerifyDiploma**: Retrieves the diploma by ID and compares the provided hash.
- **RevokeDiploma**: Sets the `isRevoked` flag and logs the revocation motif.

### Transaction Sequence Flow
```text
Frontend -> API Server -> Docker Exec (Peer) -> Chaincode (Invoke) -> Endorsement -> Commit -> Ledger
```

## 7. Database Schema

## 8. Troubleshooting
- **API 500**: Check `diplochain_user` permissions in Postgres.
- **Fabric Timeouts**: Ensure `peer0.diplochain.local` is running (`docker ps`).
- **Chaincode Registration Fail**: Verify `diplochain.go` doesn't use unsupported types (pointers, time.Time).

---

## 9. Deployment
- **Local**: Use `docker-compose.yml` and `npm run dev`.
- **Production**: Recommended to use Kubernetes (K8s) for Fabric nodes and a production-grade Nginx proxy for the frontend.
